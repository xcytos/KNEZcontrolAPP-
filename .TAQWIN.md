## Stable IDs / Serialization Rules
- session_id: `uuid4().hex` (server) or client-generated fallback; always treat as opaque string.
- message_id: stable per stored chat message; assistant placeholder pairs with user message id.
- snapshot_id: emitted by resume snapshot compiler; stored as opaque string.
- checkpoint identity: `(session_id, token_index, sha)` written to `checkpoints2`.
- MCP protocol framing: Content-Length headers with UTF-8 JSON-RPC bodies.
