## Fixed: Runtime Crash & Persistence Issues
The `TypeError: Cannot read properties of undefined (reading 'substring')` was definitively traced to `LogService` and `ChatUtils` attempting to process potentially corrupted or `null` data from previous sessions (which explains why Incognito worked).

I have:
1.  **Added Defensive Code**: Updated `LogService.ts` and `ChatUtils.ts` to strictly validate `typeof string` before substring operations.
2.  **Implemented Factory Reset**: Created a robust `ErrorBoundary.tsx` with a "Factory Reset & Restart" button that clears `localStorage` and `sessionStorage` to recover from corrupted states.
3.  **Cleaned & Rebuilt**: Performed a full `cargo clean`, removed stale binaries, and successfully ran a fresh production build (`npm run tauri build`).
4.  **Verified Launch**: The new binary `knez-control-app.exe` launches successfully.

## Verification
You should now be able to run the installed app. If you encounter the error screen again (unlikely), click the **"Factory Reset & Restart"** button to clear the old corrupted data that was causing the crash. The underlying bug that allowed the crash to happen in the first place has been patched.
