# CP6: New Subsystems & Capability Expansion Plan

Based on the analysis of the current stack (`tauri` v2, `react`, `fastapi`), here is the detailed implementation plan for Ticket Set #2 (CP6).

## 🧩 Dependencies & Scaffolding
*   **Perception (CP6-A)**: Since we are using Tauri v2, we need a way to capture the screen. The standard `tauri-plugin-shell` can invoke OS-level tools, but for a smoother experience, we'd ideally want a rust crate. However, "finishing" implies using what's available.
    *   *Plan*: Use `html2canvas` for "self-perception" (app screenshot) OR invoke a python script via KNEZ backend for desktop screenshot.
    *   *Decision*: **Backend-driven Perception**. The Control App will request a "Perception Event" from KNEZ. KNEZ (Python) has easier access to libraries like `pyautogui` or `mss` for screenshots without recompiling the Tauri Rust core (which might be fragile in this environment).
    *   *Architecture*: UI Button "Look" -> `POST /perception/snapshot` -> KNEZ captures desktop -> Returns Base64 -> UI displays it.
*   **Voice (CP6-E)**: Use standard Web Speech API (`window.speechSynthesis`, `window.webkitSpeechRecognition`) in React. No backend needed for basic version.
*   **Vector DB (CP6-D)**: Add `chromadb` or `faiss` (mocked or light) to KNEZ `memory` module.

## 🎫 Ticket Set #2 Execution Details

### 1. CP6-A: Perception Engine Integration (Backend-Driven)
*   **Objective**: Give KNEZ "eyes".
*   **Backend**: Add `knez/perception/api.py`. Endpoint `POST /perception/snapshot` using `mss` or `PIL.ImageGrab`.
*   **UI**: "Perception" tab in Chat. Button "Capture Screen". Displays image.
*   **Action**: Create `perception` module in KNEZ.

### 2. CP6-B: Automated Task Loop (Agent Mode)
*   **Objective**: Autonomous execution.
*   **Frontend**: New `AgentLoop.tsx` component.
*   **Logic**:
    1. User enters Goal.
    2. Loop: `POST /chat/completions` (Reasoning) -> Parse Tool Call -> Execute Tool -> Feed result back.
    3. Visualizer: "Thinking..." -> "Action: Search" -> "Observation: ..."

### 3. CP6-C: Multi-Window Capture (Context)
*   **Objective**: Know what user is doing.
*   **Backend**: `GET /perception/active_window`. Uses `pygetwindow` (if available) or `ctypes` on Windows.
*   **UI**: "Context Sidebar" showing "Current App: VS Code".

### 4. CP6-D: Vector Knowledge Base Management
*   **Objective**: RAG management.
*   **Backend**: `POST /memory/upload` (accepts .txt/.md). Mock vector store if `chromadb` not installed, or use simple TF-IDF for "finishing" without bloat.
*   **UI**: "Knowledge" tab in Memory Explorer. Drag-and-drop zone.

### 5. CP6-E: Voice Interface (STT/TTS)
*   **Objective**: Talk to KNEZ.
*   **Frontend**: `VoiceInput.tsx`.
*   **Logic**: `SpeechRecognition` API for input. `speechSynthesis` for reading KNEZ replies.

### 6. CP6-F: Performance Profiling Dashboard
*   **Objective**: Metrics.
*   **Frontend**: `PerformancePanel.tsx`.
*   **Data**: Mock/Real metrics from `KnezClient` interceptors (request start/end times).

### 7. CP6-G: Dark/Light Mode & Theming
*   **Objective**: Polish.
*   **Frontend**: `ThemeProvider` context. Tailwind `dark:` classes are already used, need toggle switch.

### 8. CP6-H: Local File System MCP
*   **Objective**: File access.
*   **Backend**: Create `knez/mcp/servers/filesystem.py`.
*   **Caps**: `list_dir`, `read_file`, `write_file` (restricted to `Downloads` for safety).

### 9. CP6-I: Telemetry & Usage Analytics
*   **Objective**: Stats.
*   **Frontend**: LocalStorage counter for `messages_sent`, `errors_triggered`, `sessions_created`.
*   **UI**: "Stats" modal in Settings.

### 10. CP6-J: Connection Toast Polish
*   **Objective**: UX fix.
*   **Frontend**: Update `ToastContext` to have `dismiss(id)` and `deduplicate` logic.

## 🔄 Execution Order
1.  **Backend Core** (Perception, Vector, FS MCP) -> Tickets A, C, D, H.
2.  **Frontend Core** (Agent Loop, Voice) -> Tickets B, E.
3.  **Polish** (Theming, Stats, Toasts) -> Tickets G, I, J, F.

## 🛑 Stop Condition
*   All 10 implemented.
*   Manual verification of "Capture Screen" and "Voice" (if env allows).
