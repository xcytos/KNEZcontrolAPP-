1. **CP4-A: Cognitive State Dashboard**

   * **Feature**: Visualize internal engine state (`/state/overview`, `/state/stability`).

   * **UI**: "Cognition" tab showing Focus, Stability, and active Context windows.
2. **CP4-B: Advanced Session Management (Fork & Resume)**

   * **Feature**: Utilize `POST /sessions/{id}/fork` and `resume`.

   * **UI**: "Branch Chat" button to fork a session from a specific message; "Resume" button for archived sessions.
3. **CP4-C: Operator Influence Controls**

   * **Feature**: Governance interface for `/operator/influence/*`.

   * **UI**: "Governance" panel with toggles for Global Influence, Domain rules, and Contract enforcement.
4. **CP4-D: Runbook Visualization**

   * **Feature**: Display decision trails from `/runbooks/{session_id}`.

   * **UI**: "Decision Log" side panel in Chat showing why the model made certain choices.
5. **CP4-E: Detailed Memory Inspector**

   * **Feature**: Drill-down view for `/memory/{id}`.

   * **UI**: Clicking a memory item opens a detailed modal with confidence scores, evidence references, and full content.
6. **CP4-F: MCP Registry Implementation (Backend & Frontend)**

   * **Feature**: **Backend**: Implement `/mcp/registry` in KNEZ Python server (currently missing). **Frontend**: Connect existing `McpRegistryView`.

   * **UI**: Functional "Tools" tab showing available MCP servers and tools.
7. **CP4-G: Audit & Consistency Logs**

   * **Feature**: Expose `/audit/consistency`.

   * **UI**: "System Health" section in Settings showing consistency checks and integrity status.
8. **CP4-H: Full Event Replay**

   * **Feature**: utilize `/sessions/{id}/replay` for high-fidelity session playback.

   * **UI**: "Replay Mode" in Chat to walk through a session step-by-step as it happened.
9. **CP4-I: Taqwin Context Viewer**

   * **Feature**: Display `/state/taqwin` data.

   * **UI**: Read-only view of the active `.taqwin` context currently loaded in memory.
10. **CP4-J: Contract Management**

    * **Feature**: View active contracts via `/operator/influence/contracts`.

    * **UI**: List of active behavioral contracts (e.g., "No-Code Policy", "Tone Constraints") with status.
