# PROMPT-005: Phase-2 Execution Plan

## Objective

Implement the "Cognitive Expansion" features defined in Phase-2 Planning, transitioning the Control App from a passive observer to an active reflector.

## 1. TAQWIN & Governance Updates

* **Update Task Graph**: Mark PROMPT-005 as ACTIVE.

* **Update Phase**: Transition to "Phase-2 Execution".

## 2. Domain & Data Contracts

* **Update** **`DataContracts.ts`**:

  * Add `ReflectionEntry` (Observation, Evidence, Confidence).

  * Add `MistakeEntry` (Context, Outcome, Status).

  * Add `DriftMetric` (Intent vs. Behavior).

  * Add `ChallengeEvent` (Level, Trigger, Resolution).

## 3. Feature Implementation (Group A - Reflection)

* **Unlock Sidebar**: Enable "Reflection" view in `Sidebar.tsx`.

* **Create** **`ReflectionPane`**:

  * "Analyze Session" trigger (Manual entry).

  * Structured output view (Observations with Evidence links).

  * **Invariant Check**: Read-only, no execution capability.

## 4. Feature Implementation (Group B - Mistake Ledger)

* **Update** **`MemoryExplorer`**:

  * Add "Mistakes" tab or filter.

* **Create** **`MistakeLedger`** **Component**:

  * View active/archived mistakes.

  * "Dispute" / "Acknowledge" actions (Governance surface).

  * **Invariant Check**: No silent deletion.

* (Mistake Ledger Scope Lock

  A "Mistake" is:

  * A repeated decision pattern with negative outcomes
  * Verified by evidence over time

  A "Mistake" is NOT:

  * A single failure
  * A disagreement
  * A moral judgment
  * A system opinion

  Invariant:

  * Mistake entries cannot be created without:
    * ≥2 supporting evidence references
    * Explicit confidence < 1.0)

## 5. Feature Implementation (Group C - Drift Analysis)

* **Create** **`DriftVisualizer`**:

  * Visual chart/graph of Intent vs. Behavior (Mocked data for Phase 2).

  * Embed in `ReflectionPane` or separate "Insights" view.

* (Drift Analysis Containment Rule

  * Drift metrics are visualization-only in Phase-2.
  * No drift signal may:
    * Trigger a challenge
    * Affect PresenceEngine state
    * Generate messages
  * All drift views must be labeled:
    "Exploratory visualization — not a judgment.")

## 6. Feature Implementation (Group D - Challenge System)

* **Update** **`PresenceEngine`**:

  * Add logic for "Soft Nudge" (OBSERVING -> REFLECTING).

  * Add logic for "Explicit Challenge" (Responding with Warning).

* **UI Notifications**:

  * Implement non-blocking toast/banner for Challenges.

## 7. Integration & Verification

* **Wire** **`App.tsx`**: Handle new views and state transitions.

* **Verify Governance**:

  * Ensure no autonomous actions are possible.

  * Ensure all state changes require explicit user click.

<br />

## Completion Criteria for PROMPT-005

PROMPT-005 may be marked COMPLETE only if:

* Reflection Mode produces structured, non-executive output
* Mistake Ledger is viewable but non-authoritative
* Drift Analysis is visual-only and labeled
* Challenge System cannot coerce or interrupt
* PresenceEngine remains non-autonomous
* Phase-1 guarantees remain intact

Phase-2 execution is considered "earned", not assumed.
