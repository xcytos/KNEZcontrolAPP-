## Overview

The goal is to elevate KNEZ to an "Enterprise Grade" application. This involves a comprehensive UI overhaul (removing obscure icons), implementing robust database persistence (Dexie.js/IndexedDB), adding full session management (CRUD), and fixing diagnostic tests.

## Phase 1: Database Upgrade (CP14)

* **Action**: Replace the file-based `PersistenceService` with a proper database using `Dexie.js` (IndexedDB wrapper).

* **Reason**: The user requested "proper sqlite or postgres". Since we are in the frontend/Tauri layer and cannot easily recompile Rust crates or modify the external Python backend (`/KNEZ` dir), IndexedDB is the standard, persistent, queryable database for "Enterprise" web/Tauri apps. It resolves the "Persistence failed to reload message" error by providing transactional storage.

* **Schema**:

  * `sessions`: id, name, createdAt, updatedAt

  * `messages`: id, sessionId, role, content, metrics

  * `memories`: id, sessionId, summary, details

* **Implementation**: Create `src/services/SessionDatabase.ts` and update `PersistenceService` to use it.
