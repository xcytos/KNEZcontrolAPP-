1. KNEZ MCP HTTP endpoints (code‑level)

From knez_core/app.py and knez/mcp/api.py:

Router wiring

create_app() includes mcp_api.router, so all MCP HTTP endpoints are part of the main KNEZ FastAPI app on :8000.
Config + state files

Config: _default_config_path() → repo_root / "knez-control-app" / "src-tauri" / "mcp" / "mcp.config.json".
State: _state_path() → repo_root / "knez" / "mcp_registry_state.json".
Helper functions: _load_json_file, _save_json_file, _load_registry_config, _load_registry_state.
Endpoints

GET /mcp/registry → get_mcp_registry()

Loads config via _load_registry_config():
Supports schema {"schema_version": "1", "servers": {...}} and legacy shapes.
Enforces schema_version in {"0", "1"}; otherwise 400.
Loads state from mcp_registry_state.json.
For each server key:
Reads enabled, last_error, last_ok, updated_at from state.
Produces McpItem with:
id = key
provider = "stdio"
status = "active" | "inactive" based on enabled
capabilities = ["tools", "stdio"]
Returns McpRegistryResponse(items=[...]).
GET /mcp/registry/{item_id} → get_mcp_item()

Loads config, validates schema_version as above.
Looks up item_id in servers dict; 404 if not found.
Copies entry and calls _redact_env(env):
Any env key containing KEY, TOKEN, SECRET, PASSWORD, PWD, API (case‑insensitive) is replaced with "***REDACTED***".
Returns JSON: {"id", "schema_version", "config": safe}.
POST /mcp/registry/{item_id}/toggle → toggle_mcp_item()

Validates that item_id exists in config servers; 404 if not.
_set_enabled(item_id, enabled):
Reads state JSON.
For item_id, sets enabled and updated_at = time.time().
Writes back mcp_registry_state.json.
Returns {"ok": True, "id": item_id, "enabled": enabled}.
Key reality: these endpoints are fully implemented and serve as a KNEZ‑side view and toggle of the MCP config that lives under knez-control-app/src-tauri/mcp/mcp.config.json. They are not stubs anymore.
