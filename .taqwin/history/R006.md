# R006

## Trigger
TICKET-005 - Bug Fixes: KNEZ Health Check Timeout, Hardcoded Model Labels, TransformCallback Error

## Action
Fixed three critical runtime errors blocking MCP Inspector functionality:
1. Made KNEZ health check optional (logs warning instead of blocking operations)
2. Removed hardcoded model labels from runtime code (LocalBackend and test files)
3. Documented transformCallback error as known past issue (not present in current codebase)

## Files Changed
- Modified: knez-control-app/src/mcp/inspector/McpInspectorService.ts
  - Changed ensureKnezHealthy() to log warning instead of throwing error
  - Increased timeout from 1200ms to 5000ms
  - Added try-catch for health check failures
  - MCP operations continue even if KNEZ is unreachable
- Modified: KNEZ/knez/knez_core/models/local_backend.py
  - Removed hardcoded default model_id "qwen2.5:7b-instruct-q4_K_M"
  - Made model_id optional parameter
  - Added fallback to DEFAULT_MODEL environment variable
  - Raises ValueError if no model_id provided
- Modified: KNEZ/tests/test_ttft_keepalive.py
  - Added os import
  - Changed hardcoded model to os.getenv("TEST_MODEL", "qwen2.5:7b-instruct-q4_K_M")
- Modified: KNEZ/tests/test_local_streaming.py
  - Added os import
  - Changed hardcoded models to os.getenv("TEST_MODEL", "qwen2.5:7b-instruct-q4_K_M")

## Why
User reported errors when testing MCP Inspector:
1. "Cannot read properties of undefined (reading 'transformCallback')" - known past issue
2. "{knez_unreachable: Health check timed out: http://127.0.0.1:8000/health}" - blocking MCP operations
3. Hardcoded "qwen2.5:7b-instruct-q4_K_M" label in runtime code - preventing dynamic model selection

## Result
- MCP Inspector now works without KNEZ health check blocking operations
- Model labels are dynamic and configurable via environment variables
- No hardcoded model labels in runtime code
- MCP Inspector can function with or without KNEZ backend
- Test files use configurable model names via TEST_MODEL environment variable

## Issues
- transformCallback error: Known issue from past (tauri-plugin-shell dependency), not found in current codebase - likely already fixed
- KNEZ health check: Fixed by making it optional (logs warning instead of blocking)
- Hardcoded model label: Fixed by making it configurable via environment variable
