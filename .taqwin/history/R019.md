## Agent Supervisor Features

### Agent Lifecycle
- start_agent(session_id, agent_id, task_type, parameters): Start background agent task
- stop_agent(task_id): Stop background agent task
- update_agent_progress(task_id, progress, message, metadata): Update agent progress
- _run_agent(task): Run agent task in background

### Agent Handlers
- register_agent_handler(agent_id, handler): Register handler for specific agent type
- Handlers are called when agent task starts
- Default handler is no-op (can be overridden)

### Agent Events
Emits via WebSocket system channel:
- agent_started: Agent started execution
- agent_running: Agent is running
- agent_stopping: Agent is stopping
- agent_stopped: Agent stopped execution
- agent_error: Agent encountered error
- agent_update: Agent progress update
- agent_state: Agent state change

### Task Tracking
- get_task(task_id): Get task by ID
- get_active_tasks(session_id): Get all active tasks (optionally filtered by session)
- Active tasks tracked in memory
- Tasks removed on completion or error

## Design Decision

Background agents run on backend and push updates via WebSocket. This follows the Claude/Codex pattern:
- SSE for chat streaming (tokens, tool execution)
- WebSocket for system events (agents, notifications, sync, heartbeat)

Agents never stream via SSE. All agent updates go through WebSocket system channel.
