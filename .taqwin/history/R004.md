# R004

## Trigger
Bug report: MCP Inspector hanging and stuck after test_tool click - no arguments shown

## Action
Fixed MCP Inspector tool execution - enabled actual tool calling for AI client example buttons and manual call button.

## Files Changed
- Modified: knez-control-app/src/features/mcp/inspector/McpInspectorPanel.tsx
  - Removed unused `isTesting` state
  - Added `isCallingTool` state to track tool execution
  - Made AI client example buttons actually execute tools using `svc.callTool()`
  - Enabled manual "Call Tool" button (was previously disabled)
  - Added proper error handling for tool execution
  - Updated UI message to reflect tool execution is now enabled for testing
  - Added visual feedback during tool execution ("Testing..." / "Calling...")

## Why
User reported that clicking AI client tool example buttons caused the inspector to hang with no arguments shown. The buttons were only setting the tool name and args text but not actually executing the tool. The manual call button was also disabled, preventing any tool testing.

## Result
- AI client example buttons now actually execute MCP tools with example arguments
- Manual "Call Tool" button enabled for custom tool testing
- Visual feedback during tool execution with loading state
- Proper error handling with error display
- Tool results displayed in the result area
- No more hanging or stuck behavior

## Issues
- Tool execution now enabled in inspector (previously disabled by design - ChatService owns model and MCP tool loops in production)
- This is acceptable for testing purposes in inspector
